Scores tab — strummy_eval reconstruction scoreboard

Date: 2026-07-16 · Branch: feature/eval-scores-tab (PR #44) · Original commit 2026-07-03, rebased onto main 2026-07-16

What was done

A new read-only Scores console tab (key 8) that visualizes results from the strummy_eval reconstruction harness — the local-LLM benchmark that rebuilt real Strummy modules to green against their own Jest tests. It is the results counterpart to the Evals board (which manages task lifecycle):

  • Summary strip — overall pass-rate, run count, models covered.
  • Model × task matrix — pass-rate, tok/s, iterations-to-green per cell.
  • Recent runs list — newest runs with per-run outcome.

Backend: one new guarded route GET /api/eval_scores (ingest/src/routes/eval-scores.ts). If the strummy_eval_* tables are absent it returns {"available": false} instead of 500 — the tab then renders an empty-state note. Frontend: eval-scores{-api}.js/.css + one TABS entry. Additive only; no existing behavior changed.

Context: the recurring eval cron was closed 2026-07-16 (electric bill). This tab shows the historical data already in Postgres and works for any future on-demand runs; it triggers no GPU work itself.
